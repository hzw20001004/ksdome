# 01 · 架构与数据流

## 1.1 一句话架构

阿里云 RDS 的 `jielong` 库 → Canal-Server（读 binlog）→ Canal-Adapter（写回 SQL）→ 本地 MySQL 的 `jielong` 库（整库镜像）。

## 1.2 数据流向图

```
┌─────────────────────────┐        ┌──────────────────────┐        ┌──────────────────────────┐        ┌──────────────────────┐
│   阿里云 RDS (源库)      │        │   Canal-Server       │        │   Canal-Adapter          │        │   本地 MySQL (目标)  │
│   jielong 库             │        │   读 binlog          │        │   转 SQL 写回             │        │   sig-mysql-txg      │
│                          │        │                      │        │                          │        │   jielong 库         │
│   binlog (row 格式)      │  TCP   │  解析+缓冲 (512 buffer)│  TCP  │  批量 (5000/批)           │  JDBC  │                      │
│   __#alibaba_rds_row_id# │ ─────▶ │  19 解析线程 (自动)   │ ─────▶ │  20 写入线程              │ ─────▶ │                      │
│   __ 列                  │  11111 │                      │  拉取   │                          │  3306  │                      │
└─────────────────────────┘        └──────────────────────┘        └──────────────────────────┘        └──────────────────────┘
       ▲ 启动后只读增量                    ▲ 容器名 canal-server              ▲ 容器名 canal-adapter            ▲ 容器名 sig-mysql-txg
```

## 1.3 组件清单

| 组件 | 镜像 | 容器名 | 端口 | 作用 |
|---|---|---|---|---|
| Canal-Server | `docker.1ms.run/canal/canal-server:v1.1.5` | `canal-server` | 11111 / 11112 | 伪装成 RDS 的从库，读 binlog |
| Canal-Adapter | `docker.1ms.run/slpcat/canal-adapter:v1.1.5` | `canal-adapter` | 8081 | 接收 server 数据，转 SQL 写本地 |
| 目标 MySQL | `mysql:8.0.36` | `sig-mysql-txg` | 13306→3306 | 接收同步数据的目标库 |

> **镜像说明**：
> - canal-server 用官方镜像。
> - canal-adapter **官方没有 Docker 镜像**，用社区维护的 `slpcat/canal-adapter`。备用镜像见 yml 注释。
> - 镜像源 `docker.1ms.run` 是国内加速；不通时换 `docker.m.daocloud.io`、`dockerproxy.com` 或 `registry.cn-hangzhou.aliyuncs.com`。

## 1.4 端口规划

| 端口 | 服务 | 协议 | 暴露范围 | 备注 |
|---|---|---|---|---|
| 11111 | canal-server 数据传输 | TCP（canal 私有） | 宿主机:11111 | adapter 连这个端口 |
| 11112 | canal-server metrics | HTTP | 宿主机:11112 | Prometheus 拉取用，可不开 |
| 8081 | canal-adapter REST | HTTP | 宿主机:8081 | 健康检查 / 手动重载配置 |
| 13306 | 本地 MySQL | MySQL 协议 | 宿主机:13306 → 容器 3306 | 外部访问用 13306，容器内通过服务名 `sig-mysql-txg:3306` |

## 1.5 容器依赖关系

```
sig-mysql-txg (MySQL)  ◀──────  canal-adapter  ◀──────  canal-server  ◀────── 阿里云 RDS
   独立部署                    depends_on: canal-server   独立部署
```

- MySQL 编排：`docker-compose-dev测试-mysql优化.yml`（独立）
- Canal 套件编排：`docker-compose-dev测试-Canal+CanalAdmin-优化.yml`（server + adapter，adapter `depends_on` server）
- **两个 yml 相互独立**，跨文件依赖通过 docker 网络里的服务名解析（adapter 用 `sig-mysql-txg` 连 MySQL）

> ⚠️ 同步前提：两个 compose 文件必须在 **同一个 docker 网络** 里。
> 如果分两台机器或两个独立 compose 启动，需要手动 `docker network connect` 或共享外部 network。
> 简单做法：合并到同一个 compose 启动，或者让 canal-adapter 通过宿主机 `IP:13306` 连 MySQL（要改 application.yml 的 `jdbc.url`）。

## 1.6 配置文件 → 容器内挂载点对照

| 宿主机文件（本归档） | 容器内路径 | 所属容器 |
|---|---|---|
| `canal-server-conf/canal.properties` | `/home/admin/canal-server/conf/canal.properties` | canal-server |
| `canal-server-conf/jielong-instance.properties` | `/home/admin/canal-server/conf/jielong/instance.properties` | canal-server |
| `canal-server-conf/logback.xml` | `/home/admin/canal-server/conf/logback.xml` | canal-server |
| `canal-adapter-conf/application.yml` | `/opt/canal-adapter/conf/application.yml` | canal-adapter |
| `canal-adapter-conf/rdb/jielong.yml` | `/opt/canal-adapter/conf/rdb/jielong.yml` | canal-adapter |

> 实例目录名 `jielong` 必须与 `canal.properties` 里的 `canal.destinations = jielong` 严格一致，否则 server 找不到实例配置。

## 1.7 模式说明：为什么"独立模式"而不是 canal-admin？

canal 1.1.5 起官方主推 canal-admin（Web UI 配置实例），但本次部署**没用 admin**，原因：

- 配置直接挂在文件系统，简单可控，不依赖额外 Web 服务
- 升级 / 排错时直接看文件，不用登陆 admin 后台
- 单实例场景没必要上 admin（admin 适合多实例集中管理）

关键开关：`canal.instance.global.spring.xml = classpath:spring/file-instance.xml`
这条让 server 从文件系统加载实例（而不是从 admin 拉取）。
