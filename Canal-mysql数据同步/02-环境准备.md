# 02 · 环境准备

> 上服务器后**第一件事**：完成本章。后面所有部署都依赖这一步建好的目录和文件位置。

## 2.1 主机规格要求

| 角色 | CPU | 内存 | 磁盘 | 备注 |
|---|---|---|---|---|
| 目标库机（跑 sig-mysql-txg） | ≥ 8 核（推荐 16 核） | ≥ 32G（推荐 62G） | ≥ 100G | 本次配置：62G/16核/113G |
| Canal 套件机（跑 server + adapter） | ≥ 8 核（推荐 32 核） | ≥ 16G（推荐 64G） | ≥ 50G | 本次配置：64G/32核 |

> 两套服务可以**合部**到同一台机器（最简单），也可以**分部**（库和 Canal 分开）。
> 合部时直接用 docker-compose 的服务名互相访问即可。

## 2.3 目录规划（关键）

部署根目录约定：

| 服务 | 宿主机根目录 | 容器内对应路径 |
|---|---|---|
| Canal 套件（server + adapter） | `/home/canal/` | `/home/admin/canal-server/` 和 `/opt/canal-adapter/` |
| 目标 MySQL | `/home/txg/mysql/` | `/var/lib/mysql` 等 |

### 2.3.1 创建目录（一键执行）

```bash
# Canal 套件目录
sudo mkdir -p /home/canal/canal-server-conf
sudo mkdir -p /home/canal/canal-adapter-conf/rdb
sudo mkdir -p /home/canal/canal-server-logs
sudo mkdir -p /home/canal/canal-adapter-logs

# 目标 MySQL 目录
sudo mkdir -p /home/txg/mysql/db
sudo mkdir -p /home/txg/mysql/conf
sudo mkdir -p /home/txg/mysql/data
sudo mkdir -p /home/txg/mysql/log
```

### 2.3.2 磁盘不够时：mount --bind 借空间

如果 `/home` 分区紧张（典型：根分区 50G，`/home` 分区 113G），用 `mount --bind` 把 `/mydata` 软链到 `/home`：

```bash
# 把 /home/txg 重定向到空间更大的 /mydata/txg
sudo mkdir -p /mydata/txg
sudo mount --bind /mydata/txg /home/txg

# 验证挂载
mount | grep txg
# 期望输出: /mydata/txg on /home/txg type none (rw,bind)

# 开机自动挂载 (写进 fstab)
echo "/mydata/txg /home/txg none bind 0 0" | sudo tee -a /etc/fstab
```

> 本次 MySQL 部署就是这套：113G 的 `/home` 通过 bind 给了 MySQL 数据目录。

## 2.5 Docker 网络准备

两个 compose 文件如果**在同一台机器**：

- 默认情况下 docker compose 会给每个 yml 建独立 network（名字 = 目录名_yml名_default）。
- canal-adapter 要通过服务名 `sig-mysql-txg` 访问 MySQL，**必须在同一 network**。

**方案 A（推荐，最简单）**：合并到一个目录、一个 yml 启动

把两个 yml 合并成一个 `docker-compose.yml`，server / adapter / mysql 都在里面。

**方案 B**：用外部共享 network

```bash
# 1. 创建共享网络
docker network create canal-net

# 2. 两个 yml 都加:
#    networks:
#      default:
#        name: canal-net
#        external: true
```

**方案 C**：让 canal-adapter 走宿主机 IP

修改 `canal-adapter-conf/application.yml` 的 `jdbc.url`：
```yaml
jdbc.url: jdbc:mysql://宿主机IP:13306/jielong?...   # 不再用服务名
```

## 2.6 时区与时间同步

```bash
# 宿主机时区
sudo timedatectl set-timezone Asia/Shanghai
timedatectl

# 容器时区已通过 volumes 挂载宿主机时区文件:
#   - "/etc/localtime:/etc/localtime:ro"
# 加上 environment:
#   TZ: Asia/Shanghai
# （两个 yml 里都已经配好）
```

## 2.7 防火墙 / 安全组放行

| 用途 | 端口 | 放行对象 |
|---|---|---|
| RDS 出站（canal-server → RDS 3306） | 3306 → RDS 公网/内网地址 | canal-server 容器到 RDS |
| 本地 MySQL（可选外部访问） | 13306 | 仅内网，按需 |
| canal-adapter REST（按需） | 8081 | 仅内网，按需 |

> ⚠️ **阿里云 RDS 侧**：必须把 canal-server 所在机器的出口公网 IP（或 VPC 内网 IP）加进 RDS 白名单，否则 canal-server 连不上 RDS。
